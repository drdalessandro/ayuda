# React Native UX Planner

Use this skill when UX planning needs to become concrete for the React Native Template App and its mobile-first workflows.

## Focus

This skill is for turning user journeys into app structure, screens, navigation decisions, and recurring mobile interactions.

## Design Around Mobile Realities

Keep these constraints in mind:

- sessions are short
- attention is fragmented
- notifications can help or overwhelm
- permissions and sign-in can create early friction
- users may interact while tired, stressed, or symptomatic

## Planning Areas

### Onboarding

Design the smallest onboarding flow that still builds trust.

Consider:

- welcome and positioning
- sign-in or account creation
- consent or policy acknowledgement
- baseline setup
- permission timing
- first meaningful action

### Navigation

Choose navigation based on actual product needs:

- tabs for stable daily destinations
- stack flows for setup, details, and guided tasks
- clearly separate recurring tasks from history or settings

### Daily Use

Plan what the user sees when they return:

- today’s tasks
- overdue items
- recent progress
- quick actions
- help or support access

### Notifications and Re-Engagement

Define:

- when reminders should fire
- what happens if the user ignores them
- how escalation should work
- how to avoid shame-based copy or unnecessary pressure

### Accessibility

Review:

- large text behavior
- touch target size
- VoiceOver and TalkBack support
- color contrast
- low-cognitive-load copy and layout

## Output

Produce:

- recommended navigation model
- onboarding flow
- daily-use flow
- reminder and re-engagement strategy
- accessibility priorities for the RN app

## Checklist

- [ ] Onboarding trimmed to the minimum necessary steps
- [ ] Navigation model matches core workflows
- [ ] Daily-use surface is clearly defined
- [ ] Reminder strategy avoids fatigue
- [ ] Accessibility priorities identified
